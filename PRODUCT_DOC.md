## 1. 产品概述

### 1.1 产品定位

realtime-flow 是一个面向个人投资者的 A 股资金流监测系统，实时跟踪**行业资金流向**和**个股资金流动**，通过多维度的数据分析帮助用户感知市场资金的流动规律，辅助投资决策。

### 1.2 核心价值

| 价值点 | 说明 |
|--------|------|
| **资金看得见** | 将抽象的资金流动转化为可视化的排名、图表、信号 |
| **轮动可感知** | 行业轮动强度量化，知道钱在往哪去、从哪出 |
| **异动早发现** | 主力资金突袭、顶底背离、北向资金异动等多维度预警 |
| **自选股追踪** | 关注的个股资金流一目了然，支持 UI 增删改 |
| **数据高可用** | 主源失败自动降级到新浪/腾讯，保障数据连续性 |
| **零门槛使用** | 浏览器打开即用，无需安装任何客户端 |

### 1.3 产品原则

- **数据透明**：所有数据来源可追溯，不加工不掩盖
- **轻量决策**：提供信号和建议，但不替代用户独立判断
- **持续运行**：交易时段自动采集，盘后自动静默，节假日智能识别
- **隐私安全**：所有数据存储在本地，不上传任何用户信息
- **高可用**：多数据源冗余 + 缓存回退，单点故障不影响整体可用性

## 2. 目标用户

### 2.1 主要用户画像

| 用户类型 | 特点 | 使用方式 |
|----------|------|----------|
| **个人散户** | 对 A 股有一定了解，想提升决策依据 | 每天开盘看行业轮动 + 个股评分 |
| **理财新手** | 不知道怎么选股，想感知市场热点 | 关注行业流入 TOP 和概念热度 |
| **半专业投资者** | 有自己的选股逻辑，需要数据验证 | 自选股追踪 + 背离预警 + 北向共振 |
| **量化爱好者** | 有编程能力，想基于资金流构建策略 | 通过 REST API 获取原始数据 + 回测验证 |

### 2.2 用户场景

- **早晨 9:00-9:30**：盘前预热，查看前日资金流总结
- **盘中 9:30-11:30 / 13:00-15:00**：实时监控资金异动，捕捉轮动信号
- **盘后 15:00-17:00**：复盘今日资金流，调整关注方向
- **晚间/周末**：回顾历史趋势，分析中长期资金布局，回测信号准确率

### 3.3 核心分析指标

#### 行业维度
- **行业轮动强度**：前 5 名流入总和 / 后 5 名流出总和的比值，衡量资金集中度
  - 普涨行情（无流出板块）返回 0.0 并提示"市场普涨，无明显轮动"
- **主力净流入 TOP/OUT**：按主力净额排序的行业排行榜
- **机构关注度**：超大单净流入最高的行业
- **背离预警**：主力净流入 > 1 亿但跌 > 1%（底背离），或净流出 < -1 亿但涨 > 1%（顶背离）

#### 个股维度
- **综合评分**（v2.0 优化）：
  ```
  评分 = 0.45 × 主力净占比(归一化)    # 相对值，最可靠
       + 0.25 × 主力净额(对数归一化)  # 避免大盘股霸榜
       + 0.20 × 超大单净额(归一化)
       + 0.10 × 大单净额(归一化)
  ```
- **涨跌停过滤**：主板 ±10%、科创板/创业板 ±20%、ST ±5% 的股票不参与评分
- **资金强度分级**：强流入(>5%) / 温和流入(2%~5%) / 中性(-2%~2%) / 温和流出(-2%~-5%) / 强流出(<-5%)

#### 市场维度
- **主力 vs 散户博弈**（v2.0 优化）：不再绝对化判断"健康/危险"，提示需结合位置判断
- **北向资金显著性**：北向资金日净流入超过 30 亿标记为"显著"

## 4. 用户界面

- **深色主题**：减少长时间盯盘的眼部疲劳
- **信息层级**：关键数据用大号字体 + 颜色标记，辅助信息用灰色
- **颜色语义**：绿色 = 正/流入，红色 = 负/流出，金色 = 预警/信号

## 5. 数据指标说明

### 5.1 资金流分类定义

| 资金类型 | 定义 | 含义 |
|----------|------|------|
| **超大单** | 单笔 >= 500 万元 | 机构/大资金行为 |
| **大单** | 单笔 100 万 ~ 500 万元 | 中型资金行为 |
| **中单** | 单笔 10 万 ~ 100 万元 | 中户资金行为 |
| **小单** | 单笔 < 10 万元 | 散户资金行为 |
| **主力** | 超大单 + 大单 | 市场主要参与者的资金 |

### 5.2 关键字段说明

| 字段 | 单位 | 含义 |
|------|------|------|
| 主力净流入-净额 | 亿元 | 主力资金买入额 - 卖出额 |
| 主力净流入-净占比 | % | 主力净额 / 成交额 × 100% |
| 超大单净流入-净额 | 亿元 | 超大单买入额 - 卖出额 |
| 综合评分 | 0~1 | 多维度资金流评分（v2.0 优化权重） |
| 轮动强度 | 比值 | 流入 TOP5 总和 / 流出 TOP5 总和的绝对值（普涨时为 0） |

## 6. 分析模型

### 6.1 行业轮动模型

```
输入：全行业资金流数据
过程：
  1. 按主力净流入降序排列 → 流入 TOP
  2. 按主力净流入升序排列 → 流出 TOP
  3. 按超大单净流入降序排列 → 机构关注
  4. 检测背离：主力净额正负与涨跌幅方向相反
  5. 计算轮动强度：
     - 有流出板块：sum(流入TOP5) / |sum(流出TOP5)|
     - 无流出板块（普涨）：返回 0.0，提示"市场普涨，无明显轮动"
输出：行业轮动分析报告
```

### 6.2 个股评分模型（v2.0 优化）

```
输入：个股资金流数据（已过滤涨跌停和 ST 股）
过程：
  1. 过滤涨跌停股（主板±10%/创科±20%/ST±5%）
  2. 提取 4 个因子：主力净占比、主力净额、超大单、大单
  3. 归一化：
     - 主力净占比：线性归一化到 [0,1]，区间 [-10, 10]
     - 主力净额：对数归一化（避免大盘股霸榜）
     - 超大单/大单：线性归一化
  4. 加权求和：
     评分 = 0.45×主力净占比 + 0.25×主力净额 + 0.20×超大单 + 0.10×大单
  5. 按评分降序排列
  6. 按主力占比阈值分级
输出：个股综合评分列表
```

### 6.3 信号触发规则

| 信号类型 | 触发条件 | 去重规则 |
|----------|----------|----------|
| 底背离 | 主力净流入 > 1 亿 且 涨跌幅 < -1% | 同日同板块同类型不重复 |
| 顶背离 | 主力净流入 < -1 亿 且 涨跌幅 > 1% | 同日同板块同类型不重复 |
| 北向显著 | 北向资金日净额 > 30 亿（或 < -30 亿） | 同日不重复 |
| 强流入 | 个股主力净占比 > 5% | — |
| 强流出 | 个股主力净占比 < -5% | — |
| 轮动加速 | 行业轮动强度 > 3.0 | — |

### 6.4 信号回测模型（v2.0 新增）

```
输入：历史信号 + 行业历史资金流
过程：
  1. 拉取最近 N 天的信号
  2. 对每个信号，找到发出当天的行业数据
  3. 查看信号发出后 forward_days 天的涨跌幅
  4. 判断准确性：
     - 底背离 → 预期后续上涨，实际 > 0 为正确
     - 顶背离 → 预期后续下跌，实际 < 0 为正确
  5. 统计总体准确率和分类型准确率
输出：回测报告（总数/已验证/正确数/准确率/明细）
```

### 7.2 典型投资决策辅助

| 场景 | 看什么 | 做什么 |
|------|--------|--------|

## 8. v2.0 新增能力

### 8.1 数据源冗余与降级

**问题**：v1.0 完全依赖东方财富 + akshare，接口变更或反爬升级即瘫痪。

**方案**：三级降级机制
1. **主源**：东方财富（akshare，有完整资金流明细）
2. **备用源 1**：新浪财经（批量行情，有最新价和涨跌幅）
3. **备用源 2**：腾讯财经（批量行情，有最新价和涨跌幅）

**策略**：
- 主源连续失败 3 次，自动跳过主源 5 分钟
- 降级时前端显示数据源标记 `[sina]` 或 `[tencent]`
- 主源恢复后自动切回

### 8.2 自选股 UI 管理

**问题**：v1.0 需要改 config.py 重启服务才能修改自选股。

**方案**：
- 自选股列表持久化到 SQLite `watchlist` 表
- 启动时自动从 config.py 默认值初始化
- 前端"+ 添加"按钮 + "删除"按钮
- API：`POST /api/watchlist/add`、`POST /api/watchlist/remove`、`GET /api/watchlist/config`
- 输入校验：股票代码必须 6 位数字，市场必须是 sh/sz/bj

### 8.3 信号回测验证

**问题**：v1.0 有信号无验证，不知道信号准不准。

**方案**：
- 新增 `analyzers/backtest.py` 回测引擎
- 对历史信号验证 N 日后走势准确性
- API：`GET /api/backtest/signals?days=30&forward_days=5`
- 返回总体准确率 + 分类型准确率 + 明细

### 8.4 持仓盈亏计算

**问题**：v1.0 的 `HOLDINGS` 配置存在但盈亏功能空实现。

**方案**：
- 配置持仓（代码/数量/成本）
- 实时计算市值/成本/盈亏/盈亏率
- API：`GET /api/portfolio`

### 8.5 节假日智能识别

**问题**：v1.0 不识别春节/国庆等法定假日，假日空轮询报错。

**方案**：
- 优先使用 `chinese_calendar` 库（精确到每年调休安排）
- 降级到内置简化判断（覆盖主要节假日区间）
- 假日不预热、不轮询，避免无效请求

### 8.6 体验优化

- **数据时效性**：状态栏显示"N 秒前"+ 颜色（绿/黄/红）
- **个股分页**：20 条/页，避免移动端卡顿
- **术语 tooltip**：表头悬停查看"超大单""主力占比"等定义
- **涨跌停过滤提示**：表格标题显示"ⓘ 已过滤涨跌停"

### 8.7 评分模型优化

**问题**：v1.0 用固定 ±10 亿归一化，大/小盘股失真。

**方案**：
- 主力净占比权重提升到 0.45（相对值，最可靠）
- 主力净额用对数归一化（避免大盘股霸榜）
- 过滤涨跌停股（避免虚假信号）
